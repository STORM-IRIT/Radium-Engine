4 The main renderer of Radium, implemented in the class Ra::Engine::ForwardRenderer implements a Z-pre-pass forward
5 rendering loop.
6 Even if the material association to a drawable object, realized by the so called Ra::Engine::RenderTechnique is tightly
7 coupled with the main rendering loop, Plugins might define new renderers and interact differently with shader and
8 materials properties (see chapter on [Materials](@ref develmaterials) ).
9 
10 Here is a summary of the different passes of the Ra::Engine::ForwardRenderer
11 
12 # 1. Depth, ambient color and "deferred info" pass
13 This pass
14 * only concerns opaque objects AND opaque fragment on transparent objects.
15 It is mainly used to fill in the Z-buffer, allowing to activate early z-test for next passes since the depth
16 buffer is already filled,
17 
18 * initialize the color buffer by computing e.g. the ambient color for each object,
19 
20 * generate several pictures of the scene allowing to implement composition effects later :
21  * saves the world-space normals for each object,
22  * saves the "Diffuse" aspect of the object,
23  * saves the "Specular" aspect of the object.
24 
25 
26 In this pass, each Ra::Engine::RenderObject is drawn with the Ra::Engine::Z_PREPASS argument so that
27 the corresponding shader (if it exists) will be activated before draw call.
28 (``ro->render( renderParameters, viewingParameters, DefaultRenderingPasses::Z_PREPASS );``)
29 
30 Note that the shader associated to the Ra::Engine::Z_PREPASS pass must draw only fully opaque fragments.
31 Fully transparent ones (rejected by a masking information such as mask texture) and blend-able ones
32 (those with an opacity factor alpha les than one) must be discarded.
33 
34 # 2. Lighting pass
35 This pass is a classic forward lighting pass that accumulates the color of each light source.
36 Before this pass, blending (one, one) is enabled and it does
37 ~~~text
38 for each light do
39  Get light parameters
40  for each object
41  bind shader
42  bind material & light
43  render mesh
44  done
45 done
46 ~~~
47 
48 In this pass, each RenderObject drawn with the Ra::Engine::LIGHTING_OPAQUE argument so that the
49 corresponding shader will be activated before draw call.
50 (``ro->render( renderParameters, viewingParameters, DefaultRenderingPasses::LIGHTING_OPAQUE );``)
51 
52 Note that the shader associated to the Ra::Engine::LIGHTING_OPAQUE pass must lit and draw only fully opaque fragments.
53 Fully transparent ones (rejected by a masking information such as mask texture) and blend-able ones
54 (those with an opacity factor alpha les than one) must be discarded.
55 
56 # 3. Ordered independent transparency
57 Rendering transparent objects in Radium is done according to the algorithm described in
58 * Weighted Blended Order-Independent Transparency,
59  Morgan McGuire, Louis Bavoil - NVIDIA,
60  Journal of Computer Graphics Techniques (JCGT), vol. 2, no. 2, 122-141, 2013,
61  <http://jcgt.org/published/0002/02/09/>
62 
63 This pass contains one scene rendering pass and one composition pass.
64 
65 * The scene rendering pass must compute both the accumulation buffer and the coverage buffer as described in the paper
66 (see the Material documentation for example of shader.). It is realized the same way than the lighting pass but only
67 fragments that are transparent must be lit and drawn.
68 
69 * The composition pass then adds to the color buffer the resulting blended color.
70 
71 # 4. Post-process the whole _render pass_
72 This pass takes the color buffer, representing colors in linear RGB space) and apply gamma correction to the image
73 
74 # 5. Debug and UI render objects
75 Those objects are drawn with their own shader, and without lighting.
76 They do not write in the depth map but they have different depth testing behaviour :
77 * func is `LESS` for debug objects (drawn only if visible)
78 * func is `ALWAYS` for UI objects (drawn in front of everything else)
79 
80 # 6. Write final texture to framebuffer / backbuffer
81 This method is just responsible for displaying the final stuff on screen or on the saved FBO.

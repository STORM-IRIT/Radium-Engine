1 # Timeline And Keyframes
2 
3 The `Ra::GuiBase::Timeline` class provides display and management of `Ra::Core::Animation::KeyFramedValue`s through
4 the `Ra::Core::Animation::KeyFramedValueController` class.
5 
6 ## The Timeline UI
7 
8 The Timeline UI is a widget allowing to manipulate time and display animation keyframes:
9 ![Timeline UI](../images/timeline_ui.png)
10 The central part of the UI shows the time space, in which we have:
11  - the time scale on the bottom part,
12  - the time window drawn as a green to red sheet between movable widgets defining its start-time and end-time,
13  - the current time drawn as a blue vertical line,
14  - the time values, drawn as yellow vertical lines, where a keyframe is defined for the selected KeyFramedValue.
15 
16 On the upper-left corner, one can define the time window for time flow, i.e.
17  - define the start-time and end-time for time flow,
18  - change the current time value,
19  - change the maximal time display.
20 
21 On the upper-middle part, one can manipulate time flow, i.e.
22  - reset time to the start-time or the end-time,
23  - get to the previous or next keyframe time,
24  - make time flow,
25  - manage the time flow policy (loop-around or forward-backward).
26 
27 On the upper-right part, one can manipulate animation keyframes, i.e.
28  - choose the one to display and manipulate for the selected object,
29  - add a new keyframe or update the one at the current time,
30  - see the current number of keyframes,
31  - remove the current keyframe,
32  - [coming soon] open the keyframe editor.
33 
34 On the upper-far-right corner, one can open the help window, displaying shortcuts and hints on how to manipulate
35 the time space and the selected KeyFramedValue:
36 
37 ![Timeline Help](../images/timeline_help.png)

39 ## Display animated data using the Timeline
40 
41 `Ra::Core::Animation::KeyFramedValueController`s must be registered into the `Ra::GuiBase::Timeline`, which can be done using the
42 `Ra::GuiBase::Timeline::registerKeyFramedValue` methods, binding them to the `Ra::Engine::Entity`, `Ra::Engine::Component` or
43 `Ra::Engine::RenderObject` they belong to.
44 
45 ### Binding "Static" KeyFramedValue
46 Those are `Ra::Core::Animation::KeyFramedValue`s that are an explicit part of a `Ra::Engine::Entity`, `Ra::Engine::Component` or
47 `Ra::Engine::RenderObject` data, either filled upon construction or through the `Ra::GuiBase::Timeline`.
48 Static `Ra::Core::Animation::KeyFramedValue`s must be registered in the `Ra::GuiBase::Timeline` after the object's
49 construction.
50 They usually are not bound to an *UpdateCallback* function since the object they
51 belong to usually calls `Ra::Core::Animation::KeyFramedValue::at` to query the current value.
52 
53 Example Usage:
54 ```c++
55 using KeyFramedValue = Ra::Core::Animation::KeyFramedValue<Scalar>;
56 using KeyFramedValueController = Ra::Core::Animation::KeyFramedValueController;
57 
58 struct MyComponentWithKeyFrame : public Ra::Engine::Component
59 {
60  MyComponentWithKeyFrame( const std::string& name, Ra::Engine::Entity* entity, Scalar value = 0_ra )
61  : Ra::Engine::Component( name, entity )
62  , m_keyframes( value, 0_ra )
63  , m_currentValue( value )
64  {}
65 
66  void initialize() override {}
67  void setTime( Scalar t ) {
68  // Here we use the value at time t using linear interpolation between keyframes.
69  m_currentValue = m_keyframes.at( t, Ra::Core::Animation::linearInterpolate<Scalar> );
70  // Then update other data that may depend on that new value.
71  update();
72  }
73 
74  private:
75  void update() {
76  // Here we can update other data that depend on the value of m_currentValue.
77  }
78 
79  KeyFramedValue m_keyframes;
80  Scalar m_currentValue;
81 };

90  void AnimationSystem::handleAssetLoading( Ra::Engine::Entity* entity, const Ra::Core::Asset::FileData* fileData ) {
91  // Add a Component whenever a file is loaded
92  auto comp = new MyComponentWithKeyFrame( "MyComponentWithKeyFrame_", entity );
93  registerComponent( entity, comp );
94  if ( m_timeline )
95  {
96  // on insertion we may want to get the frame value from elsewhere
97  auto inserter = [comp]( const Scalar& t ) {
98  // for example value(t) + 1
99  comp->m_keyframes.insertKeyFrame( t, comp->m_keyframes.at( t ) + 1_ra );
100  };
101  // No need for an update callback since the component owns the KeyFramedValue.
102  m_timeline->registerKeyFramedValue( comp, KeyFramedValueController( &comp->m_keyframes, "KeyFrame", inserter ) );
103  }
104  }

122 ### Binding "Dynamic" KeyFramedValues:
123 Those are `Ra::Core::Animation::KeyFramedValue`s that are not part of an `Ra::Engine::Entity`, `Ra::Engine::Component` or
124 `Ra::Engine::RenderObject` data, but are used to keyframe some of its data.
125 Dynamic `Ra::Core::Animation::KeyFramedValue`s must be created for and owned by the UI, and registered in the
126 `Ra::GuiBase::Timeline`.
127 They are usually bound to an UpdateCallback function since they have to update the
128 object's data they are linked to.
129 
130 Example Usage:
131 ```c++
132 /// Let's say there is a Component class defined as:
133 struct MyComponent : public Component {
134  MyComponent( const std::string& name, Ra::Engine::Entity* entity, Scalar value = 0_ra )
135  : Ra::Engine::Component( name, entity )
136  , m_currentValue( value )
137  {}
138 
139  void initialize() override {}
140  void update() { std::cout << m_currentValue << std::endl; }
141 
142  Scalar m_currentValue;
143 };

145 /// We here create a plugin that will keyframe the MyComponent::m_currentValue
146 /// attribute of selected MyComponents in the scene:
147 
148 // Let's start with some simple UI:
149 struct MyWidget : public QWidget {
150  Q_OBJECT
151 
152  MyWidget( QWigdet* parent = nulltpr ) : QWidget( parent ) {
153  m_keyFrameValueCheckbox = new QCheckBox( "Keyframe current Value", this );
154  }
155 
156  QCheckBox* m_keyFrameValueCheckbox;
157 };
158 
159 // Then the plugin:
160 struct MyPlugin : public QObject, Ra::Plugins::RadiumPluginInterface {
161  Q_OBJECT
162  Q_RADIUM_PLUGIN_METADATA
163  Q_INTERFACES( Ra::Plugins::RadiumPluginInterface )
164 
165  MyPlugin() = default;
166  ~MyPlugin() {
167  for ( auto kf : m_keyframes )
168  { delete kf.second; }
169  }
170 
171  void registerPlugin( const Ra::Plugins::Context& context ) override {
172  // get timeline
173  m_timeline = context.m_timeline;
174  // connect to selection manager
175  m_selectionManager = context.m_selectionManager;
176  if ( m_selectionManager )
177  {
178  connect( m_selectionManager, &Ra::GuiBase::SelectionManager::currentChanged, this, &MyPlugin::onCurrentChanged );
179  }
180  }
181  bool doAddWidget( QString& name ) override {
182  return true;
183  }
184  QWidget* getWidget() override {
185  m_widget = new MyWidget;
186  connect( m_widget->m_keyFrameValueCheckbox, &QCheckBox::toggled, this, &MyPlugin::keyFrameValue );
187  return m_widget;
188  }
189  bool doAddMenu() override { return false; }
190  QMenu* getMenu() override { return nullptr; }
191  bool doAddAction( int& nb ) override { return false; }
192  QAction* getAction( int id ) override { return nullptr; }
193  void onCurrentChanged( const QModelIndex& current, const QModelIndex& prev ) {
194  // deal with selection
195  if ( m_selectionManager->hasSelection() )
196  {
197  const Ra::Engine::ItemEntry& ent = m_selectionManager->currentItem();
198  m_current = dynamic_cast<MyComponent*>( ent.m_component );
199  }
200  else
201  {
202  m_current = nullptr;
203  }
204  m_widget->m_keyFrameValueCheckbox->setChecked( m_keyframes.find( m_current ) != m_keyframes.end() );
205  }
206  void keyFrameValue( bool on ) {
207  // deal with keyframing
208  if ( m_current == nullptr ) { return; }
209  if ( on )
210  {
211  // keyframe m_current->m_currentValue
212  auto keyframes = new KeyFramedValue( m_current->m_currentValue, 0, interpolate );
213  // on insertion we may want to get the frame value from elsewhere
214  auto inserter = [keyframes]( const Scalar& t ) {
215  // for example value(t) + 1
216  keyframes->insertKeyFrame( t, keyframes->at( t ) + 1_ra );
217  };
218  // on update we here want to modify the value since the MyComponent cannot do it by itself
219  Scalar& value = &m_current->m_currentValue;
220  auto updater = [keyframes, &value]( const Scalar& t ) {
221  value = keyframes->at( t );
222  };
223  // register the new KeyFramedValue
224  m_keyframes.push_back( keyframes );
225  m_timeline->registerKeyFramedValue( m_current, KeyFramedValueController( keyframes, "KeyFrame", inserter, updater ) );
226  }
227  else
228  {
229  // release m_current->m_currentValue from the keyframe
230  m_timeline->unregisterKeyFramedValue( m_current, "KeyFrame" );
231  }
232  };
233 
234  MyWidget* m_widget{nullptr};
235  Ra::GuiBase::Timeline* m_timeline{nullptr};
236  MyComponent* m_current;
237  std::map<MyComponent*,KeyFramedValue*> m_keyframes;
238 };

4 # Dependencies
5 All the dependencies are automatically fetched and compiled at build time.
6  * [Core] Eigen, OpenMesh
7  * [Engine] glm, globjects, glbindings
8  * [IO] Assimp
9  * [GuiBase] Qt Core, Qt Widgets and Qt OpenGL v5.5+ (5.10 recommended)
10  * stb_image
11 
12 See @ref develbuildchain for technical details on dependencies management and how to provide locally installed dependencies.
13 
14 Minimal requirements
15  * OpenGL 3+ / GLSL 330+
16  * CMake 3.0.+

18 # Supported compiler and platforms

61 ## Configure build
62 
63 Radium offers the following build options:
64 ~~~bash
65 // Enable testing. Tests are automatically built with target all
66 RADIUM_ENABLE_TESTING:BOOL=ON
67 --
68 // Include Radium::Core in CMake project
69 RADIUM_GENERATE_LIB_CORE:BOOL=ON
70 --
71 // Include Radium::Engine in CMake project
72 RADIUM_GENERATE_LIB_ENGINE:BOOL=ON
73 --
74 // Include Radium::GuiBase in CMake project
75 RADIUM_GENERATE_LIB_GUIBASE:BOOL=ON
76 --
77 // Include Radium::IO in CMake project
78 RADIUM_GENERATE_LIB_IO:BOOL=ON
79 --
80 // Include Radium::PluginBase in CMake project
81 RADIUM_GENERATE_LIB_PLUGINBASE:BOOL=ON
82 --
83 // Check submodules during build (will be automatically disabled after run)
84 RADIUM_GIT_UPDATE_SUBMODULE:BOOL=ON
85 --
86 // Install documentation. If RadiumDoc is compiled, install documentation to bundle directory for install target
87 RADIUM_INSTALL_DOC:BOOL=ON
88 --
89 // Value of CMAKE_INSTALL_MESSAGE for dependencies. See documentations of CMAKE_INSTALL_MESSAGE for possible values
90 RADIUM_EXTERNAL_CMAKE_INSTALL_MESSAGE=NEVER
91 --
92 // Disable Radium Log messages
93 RADIUM_QUIET:BOOL=OFF
94 --
95 // Provide loaders based on Assimp library
96 RADIUM_IO_ASSIMP:BOOL=OFF
97 --
98 // Provide depricated loaders (to be removed without notice)
99 RADIUM_IO_DEPRECATED:BOOL=ON
100 --
101 // Provide loaders based on TinyPly library
102 RADIUM_IO_TINYPLY:BOOL=ON
103 --
104 // [addExternalFolder] Skip updating Core::external (disable for rebuild)
105 RADIUM_SKIP_CORE_EXTERNAL:BOOL=ON
106 --
107 // [addExternalFolder] Skip updating Engine::external (disable for rebuild)
108 RADIUM_SKIP_ENGINE_EXTERNAL:BOOL=ON
109 --
110 // [addExternalFolder] Skip updating IO::external (disable for rebuild)
111 RADIUM_SKIP_IO_EXTERNAL:BOOL=ON
112 --
113 // Update version file each time the project is compiled (update compilation time in version.cpp)
114 RADIUM_UPDATE_VERSION:BOOL=ON
115 ~~~
116 
117 All radium related cmake options (with their current values) can be printed with `cmake -LAH | grep -B1 RADIUM` (on linux like system)

119 ## Building on Linux/MacOS (command line instruction)
120 
121 Out-of source builds are mandatory, we recommand to follow the usual sequence:
122 
123 ~~~bash
124 $ mkdir build
125 $ cd build
126 $ cmake ..
127 $ make
128 ~~~
129 
130 Finally, the `install` target will copy all the radium related library in the same place, usefull for App compilation
131 
132 ~~~bash
133 $ make install
134 ~~~
135 
136 Note that installation requires write access on the installation directory.

138 ## Building on Microsoft Windows with Visual Studio
139 
140 ### Supported versions of MSVC
141 Radium requires MSVC 2019 or superior, as it relies on:
142 * C++11/C++14/C++17 features such as `constexpr`,
143 * cmake built-in support
144 
145 Our Continuous Integration systems uses Microsoft Compiler 2017, in combination with cmake and ninja.
146 Using Visual Studio 2017 with cmake support is however not possible: VS is shipped with cmake: 3.12, while Radium requires cmake 3.13 at least. We recommand to use Visual Studio 2019 in that case.
147 Qt 5.15+ is distributed with binaries precompiled with MSVC 2019, but Qt binaries precompiled with MSVC2017 does not break the build.
148 
149 ### Dependencies
150 
151 *Qt* distributes precompiled libraries for VS 2017 - 64 bits (tested with 5.10 and 5.12).
152 If using earlier versions of Qt (5.5) or a different toolset you may have to compile Qt yourself.
153 You will probaby have to manually point cmake to the Qt folder.
154 
155 Other dependencies (Eigen, Assimp, glbinding, globjects and glm) are fetched and compiled at configure-time if they are not present at this time on the host system (see @ref develbuildchain).

158 ### Getting started with Visual Studio
159 
160 Thanks to the integrated support of CMake in Visual Studio, you don't need a VS solution to build your project: open the Radium folder (via *File* > *Open* > *Folder ...* or `devenv.exe <foldername>`).
161 VS should run cmake, generate the target builds (Debug and Release by default).
162 Other build types can be added by editing `CMakeSettings.json`.
163 
164 You may have Cmake errors occuring at the first run.
165 To fix them, you need to edit the VS-specific file `CMakeSettings.json`, via *CMake* > *Change CMake Settings* > path-to-CMakeLists (configuration-name) from the main menu.
166 For instance, it usually requires to set cmake build types manually, and to give path to Qt libraries.
167 To fix it, edit `CMakeSettings.json`, such that
168 ~~~json
169 {
170  "configurations": [
171  {
172  "name": "x64-Release",
173  "generator": "Ninja",
174  "configurationType": "Release",
175  "inheritEnvironments": [ "msvc_x64_x64" ],
176  "buildRoot": "C:/Users/XXX/Dev/builds/Radium/${name}",
177  "installRoot": "C:/Users/XXX/Dev/Radium-install",
178  "cmakeCommandArgs": "-DCMAKE_PREFIX_PATH=C:/Qt-5.10/5.10.0/msvc2017_64 -DRADIUM_UPDATE_VERSION=OFF -DRADIUM_IO_ASSIMP=ON",
179  "buildCommandArgs": "",
180  "ctestCommandArgs": ""
181  },
182  {
183  "name": "x64-Debug",
184  "generator": "Ninja",
185  "configurationType": "Debug",
186  "inheritEnvironments": [ "msvc_x64_x64" ],
187  "buildRoot": "C:/Users/XXX/Dev/builds/Radium/${name}",
188  "installRoot": "C:/Users/XXX/Dev/Radium-installdbg",
189  "cmakeCommandArgs": "-DCMAKE_PREFIX_PATH=C:/Qt-5.10/5.10.0/msvc2017_64 -DRADIUM_UPDATE_VERSION=OFF -DRADIUM_IO_ASSIMP=ON",
190  "buildCommandArgs": "",
191  "ctestCommandArgs": ""
192  }
193  ]
194 }
195 ~~~
196 *Note*: it is strongly encouraged to use `/` separators in your path, instead of `\\` as previously mentionned. See https://stackoverflow.com/questions/13737370/cmake-error-invalid-escape-sequence-u
197 
198 
199 *Note*: When compiling the dependencies you may hit the max path length fixed by Microsoft OS (260 characters). To fix this, you might need to change the path of your build dir to shorten it, or to change the limit on your system, see: https://docs.microsoft.com/en-us/windows/win32/fileio/naming-a-file#enable-long-paths-in-windows-10-version-1607-and-later
